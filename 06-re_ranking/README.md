## 🎯 하이브리드 검색(RRF)의 한계와 Re-Ranking의 필요성

### 하이브리드 검색(RRF)의 한계점

하이브리드 검색은 BM25(키워드)와 Vector(벡터) 검색의 결과를 **RRF(Reciprocal Rank Fusion)**로 결합합니다. 하지만 RRF는 **단순 순위 결합** 알고리즘입니다.

```
❌ 하이브리드 검색(RRF)의 한계:

1. 순위만 결합, 의미는 모름
   - BM25 1위 + Vector 3위 → RRF 점수 계산
   - 실제로 쿼리 의도에 맞는지는 알 수 없음

2. 문서 내용과 쿼리의 "진짜 관련성" 평가 불가
   - "따뜻하고 가벼운 자켓" 검색 시
   - 키워드 매칭: "자켓" 포함된 상품
   - 벡터 유사도: 자켓과 비슷한 임베딩
   - 하지만 리뷰에 "실제로 따뜻해요"가 있는지는 모름

3. 복합적인 쿼리 의도 파악 어려움
   - "아이 생일 선물용으로 좋은 유아용품"
   - RRF는 각 단어/벡터 매칭의 순위만 결합
   - "선물용으로 좋은"이라는 의도를 이해하지 못함
```

### Semantic Re-Ranking이 해결하는 문제

**Semantic Re-Ranking**은 하이브리드 검색 결과(상위 50개)를 Microsoft의 **언어 모델**로 다시 평가하여 **쿼리와 문서의 실제 의미적 관련성**을 기준으로 재정렬합니다.

```
검색 흐름:
┌──────────────────┐     ┌──────────────────┐
│ 1. BM25 검색      │     │ 2. Vector 검색    │
│ (키워드 매칭)     │     │ (임베딩 유사도)   │
└────────┬─────────┘     └────────┬─────────┘
         │                         │
         └────────┬────────────────┘
                  ↓
         ┌──────────────────┐
         │ 3. RRF 결합       │ → 상위 50개 결과 (순위 기반 결합)
         └────────┬─────────┘
                  ↓
         ┌──────────────────┐
         │ 4. L2 Re-Rank    │ → 언어 모델로 의미적 관련성 재평가
         └────────┬─────────┘   (쿼리 ↔ 문서 내용 직접 비교)
                  ↓
         ┌──────────────────┐
         │ 5. Captions      │ → 관련 문장 자동 추출
         └──────────────────┘
```

## 🛠️ 실습 구조

이번 실습은 **기존 인덱스를 활용**하여 Semantic Configuration만 추가합니다:

```
[이전 실습]
02-keyword_search → 인덱스 생성, 데이터 업로드
03-vector_search → 벡터 필드 추가
04-hybrid_search → 하이브리드 검색 실습 (RRF)

[이번 실습]
06-re_ranking → Semantic Configuration 추가
                ↓
        하이브리드 검색 (BM25 + Vector + RRF)
                ↓
        + L2 Re-Ranking (의미적 재정렬)
```

**⚠️ 중요**: Semantic Re-Ranking은 **하이브리드 검색 결과를 재정렬**하는 것입니다.
- 키워드 검색의 대체 ❌
- 벡터 검색의 대체 ❌
- **하이브리드 검색 결과의 품질 향상** ✅

## 📊 5가지 비즈니스 시나리오

### 시나리오 1: 리뷰 기반 품질 검색
**비즈니스 상황**: "따뜻하고 가벼운 자켓" - 실제 그런 평가를 받은 상품이 상위에 와야 함

| 검색 모드 | 동작 | 한계/장점 |
|----------|------|----------|
| **Hybrid(RRF)** | '따뜻', '가벼운' 키워드+벡터 순위 결합 | 리뷰 내용의 실제 의미 모름 |
| **+ Re-Ranking** | 리뷰에서 "따뜻해요", "가벼워요" 실제 평가 이해 | 긍정 평가 상품 상위 노출 |

### 시나리오 2: 복합 의도 파악
**비즈니스 상황**: "아이 생일 선물용으로 좋은 제품" - "선물용으로 좋은" 의미 파악 필요

| 검색 모드 | 동작 | 한계/장점 |
|----------|------|----------|
| **Hybrid(RRF)** | '아이', '생일', '선물' 각각의 순위 결합 | 전체 의도 이해 못함 |
| **+ Re-Ranking** | "선물용으로 좋아요" 같은 맥락 이해 | 선물 적합성 평가된 상품 우선 |

### 시나리오 3: 상품 특성 기반 검색
**비즈니스 상황**: "활동적인 운동할 때 입기 좋은 옷" - 실제 사용감 평가 필요

| 검색 모드 | 동작 | 한계/장점 |
|----------|------|----------|
| **Hybrid(RRF)** | '활동', '운동' 키워드+벡터 순위 결합 | 실제 착용감 평가 모름 |
| **+ Re-Ranking** | "신축성 좋아요", "운동할 때 편해요" 이해 | 실제 사용 후기 반영 |

### 시나리오 4: 품질/만족도 기반 검색
**비즈니스 상황**: "품질 좋고 만족도 높은 제품" - 실제 고객 평가 이해 필요

| 검색 모드 | 동작 | 한계/장점 |
|----------|------|----------|
| **Hybrid(RRF)** | '품질', '만족도' 키워드+벡터 순위 결합 | '품질' 단어만 매칭 |
| **+ Re-Ranking** | "품질이 좋아요", "만족스러워요" 실제 평가 이해 | 긍정 평가 상품 우선 |

### 시나리오 5: 용도/목적 기반 검색
**비즈니스 상황**: "데일리로 매일 입기 좋은 편한 옷" - 일상 착용 적합성 평가 필요

| 검색 모드 | 동작 | 한계/장점 |
|----------|------|----------|
| **Hybrid(RRF)** | '데일리', '매일', '편한' 순위 결합 | 실제 사용 패턴 모름 |
| **+ Re-Ranking** | "매일 입어요", "편하게 잘 입고 있어요" 이해 | 일상 착용 적합 상품 우선 |

## 🔧 Semantic Configuration 구성

Re-Ranking 시 언어 모델이 참고할 필드를 지정합니다:

| 필드 유형 | 역할 | 우리 데이터셋 |
|----------|------|--------------|
| **titleField** | 제목 (가중치 높음) | `title` |
| **contentFields** | 본문 내용 | `review`, `brand` |
| **keywordFields** | 키워드/태그 | `category` |

### 필드별 가중치 (자동 적용)

```
titleField:      가중치 ⭐⭐⭐⭐⭐ (매우 높음)
contentFields:   가중치 ⭐⭐⭐ (중간) → 리뷰 내용 분석에 중요!
keywordFields:   가중치 ⭐⭐ (낮음)
```

## 💡 주요 기능

### 1. Semantic Captions
쿼리와 가장 관련 있는 **문장을 자동 추출**하여 하이라이트 제공

```python
# 예시: "따뜻한 자켓" 검색
Caption: "보온성이 뛰어나 <em>따뜻하게</em> 입을 수 있어요"
```

### 2. Semantic Answers
질문형 쿼리에 대해 **직접 답변을 추출**

```python
# 예시: "어떤 자켓이 따뜻한가요?"
Answer: "충전재가 두껍고 보온성이 뛰어나 겨울에도 따뜻합니다"
```

### 3. @search.rerankerScore
각 문서의 **쿼리 의도와의 실제 관련성 점수** (0~4 범위)

## ⚠️ 주의사항

### 1. 비용 고려
- Semantic Search는 **별도 요금 부과** (Standard: 쿼리당 $0.50/1000건)
- 필요한 쿼리에만 선택적 사용 권장

### 2. 최대 제한
- Re-ranking 대상: **최대 50개 문서**
- 하이브리드 검색 상위 50개 결과만 재정렬됨

### 3. 응답 시간
- 하이브리드 검색 대비 +50~100ms 추가 (언어 모델 처리)

## 🎯 언제 Re-Ranking을 사용하나?

| 상황 | Hybrid(RRF)만 | + Re-Ranking |
|------|--------------|--------------|
| 정확한 제품명 검색 | ✅ 충분 | 불필요 |
| 리뷰 기반 품질 검색 | 한계 있음 | ✅ 권장 |
| 복합 의도 쿼리 | 한계 있음 | ✅ 권장 |
| 단순 필터 검색 | ✅ 충분 | 불필요 |
| 비용 민감한 대량 검색 | ✅ 권장 | 선택적 |

## 💡 핵심 요약

```
하이브리드 검색(RRF) = 키워드 매칭 순위 + 벡터 유사도 순위 결합
                       ↓
            "순위"만 결합, 실제 의미적 관련성은 모름
                       ↓
            Re-Ranking = 언어 모델로 쿼리 ↔ 문서 의미 재평가
                       ↓
            쿼리 의도에 더 정확히 부합하는 결과 상위 노출
```
