## 1. AI Enrichment란 무엇인가?

### 1.1 정의

**AI Enrichment(AI 보강)**는 Azure AI Search에서 **원본 데이터를 검색 가능한 형태로 변환하는 AI 기반 처리 파이프라인**입니다.

쉽게 말해, **검색하기 어려운 형태의 콘텐츠**(이미지, PDF 내 스캔 문서, 비정형 텍스트 등)를 **AI를 통해 분석하고 추출하여 검색 가능한 구조화된 데이터로 변환**하는 기능입니다.

### 1.2 핵심 아이디어

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI Enrichment 파이프라인                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   [원본 데이터]        [AI 처리]           [결과물]               │
│                                                                  │
│   📄 스캔된 PDF   →   🔍 OCR 처리    →   📝 검색 가능한 텍스트    │
│   🖼️ 상품 이미지  →   🤖 이미지 분석 →   🏷️ 자동 태그/설명       │
│   📝 고객 리뷰    →   💬 감정 분석   →   😊😐😢 감정 점수        │
│   📚 문서 더미    →   🔑 핵심구문추출 →   📋 키워드 목록          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 왜 필요한가?

일반적인 검색 엔진은 **텍스트**만 검색할 수 있습니다. 하지만 실제 비즈니스 데이터는 다양한 형태로 존재합니다:

| 데이터 유형 | 검색 문제점 | AI Enrichment 해결책 |
|------------|-----------|---------------------|
| 스캔된 문서/이미지 | 텍스트가 없음 | OCR로 텍스트 추출 |
| 다국어 콘텐츠 | 언어별 검색 필요 | 언어 감지 + 번역 |
| 이미지 파일 | 메타데이터만 검색 가능 | 이미지 분석으로 설명 생성 |
| 긴 문서 | 전체 내용 파악 어려움 | 핵심 구문/엔티티 추출 |
| 고객 피드백 | 정성적 데이터 분류 어려움 | 감정 분석 |

## 2. Skillset의 핵심 개념

### 2.1 Skillset이란?

**Skillset**은 **하나 이상의 AI 스킬(Skill)을 묶은 집합**입니다. Indexer와 연결되어 데이터가 인덱싱되기 전에 AI 처리를 수행합니다.

```
┌──────────────────────────────────────────────────────────────┐
│                       데이터 처리 흐름                         │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│   [Data Source]  →  [Indexer]  →  [Skillset]  →  [Index]     │
│      (Blob)           │            │                         │
│                       │         ┌──┴──┐                      │
│                       │         │Skill│ (OCR)                │
│                       │         │Skill│ (언어감지)            │
│                       │         │Skill│ (번역)               │
│                       │         │Skill│ (감정분석)            │
│                       │         └─────┘                      │
│                       │            │                         │
│                       │            ↓                         │
│                       │    [Knowledge Store]                 │
│                       │       (선택사항)                      │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 Enrichment Tree (보강 트리)

Skillset 실행 중 생성되는 **임시 트리 구조의 데이터**입니다. 각 스킬은 이 트리에서 데이터를 읽고, 새로운 노드를 추가합니다.

```
/document                          ← 문서 루트
    ├── /document/content          ← 원본 텍스트
    ├── /document/normalized_images/*  ← 추출된 이미지들
    ├── /document/language         ← 감지된 언어 (스킬 출력)
    ├── /document/keyphrases       ← 핵심 구문 (스킬 출력)
    ├── /document/entities         ← 인식된 엔티티 (스킬 출력)
    │       ├── persons
    │       ├── organizations
    │       └── locations
    └── /document/sentiment        ← 감정 점수 (스킬 출력)
```

### 2.3 Context (컨텍스트)

각 스킬이 **어느 범위에서 실행될지** 결정합니다:

| Context | 의미 | 실행 횟수 |
|---------|------|----------|
| `/document` | 문서 전체 | 문서당 1회 |
| `/document/pages/*` | 각 페이지 | 페이지 수만큼 |
| `/document/normalized_images/*` | 각 이미지 | 이미지 수만큼 |

## 3. Built-in Skills 종류

Azure AI Search에서 제공하는 **기본 제공 스킬**들입니다. Azure AI Services(Cognitive Services)와 연동됩니다.

### 3.1 텍스트 처리 스킬

| 스킬 | 기능 | 활용 예시 |
|-----|------|---------|
| **Language Detection** | 텍스트 언어 자동 감지 | 다국어 콘텐츠 분류 |
| **Text Translation** | 텍스트 번역 | 외국어 문서 한국어 변환 |
| **Text Split** | 긴 텍스트를 청크로 분할 | 벡터 임베딩을 위한 전처리 |
| **Key Phrase Extraction** | 핵심 키워드 추출 | 문서 자동 태깅 |
| **Entity Recognition** | 사람, 조직, 장소 등 인식 | 계약서에서 회사명 추출 |
| **Sentiment Analysis** | 긍정/부정/중립 감정 분석 | 고객 리뷰 분류 |
| **PII Detection** | 개인정보(이름, 전화번호 등) 탐지 | 민감정보 마스킹 |

### 3.2 이미지 처리 스킬

| 스킬 | 기능 | 활용 예시 |
|-----|------|---------|
| **OCR (Optical Character Recognition)** | 이미지에서 텍스트 추출 | 스캔된 문서 검색 |
| **Image Analysis** | 이미지 설명, 태그, 카테고리 생성 | 상품 이미지 자동 분류 |
| **Document Layout** | 문서 구조(표, 단락) 분석 | 복잡한 문서 파싱 |

### 3.3 유틸리티 스킬

| 스킬 | 기능 | 활용 예시 |
|-----|------|---------|
| **Shaper** | 데이터 형태 재구성 | Knowledge Store 출력 형식 지정 |
| **Merge** | 여러 필드 병합 | OCR 결과 + 원본 텍스트 결합 |
| **Conditional** | 조건부 로직 | 특정 조건일 때만 처리 |

## 4. Custom Skills

### 4.1 Custom Skill이란?

Built-in Skill로 처리할 수 없는 **비즈니스 고유 로직**을 외부 API로 구현하여 연결하는 방식입니다.

### 4.2 구현 방식

```
┌─────────────────────────────────────────────────────────────┐
│                    Custom Skill 아키텍처                     │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   [Azure AI Search]                                          │
│         │                                                    │
│         │  HTTP POST (JSON)                                  │
│         ▼                                                    │
│   ┌─────────────────┐                                        │
│   │ Azure Function  │  ← 또는 Web App, Container 등          │
│   │ (Custom Logic)  │                                        │
│   └─────────────────┘                                        │
│         │                                                    │
│         │  외부 서비스 호출 가능                               │
│         ▼                                                    │
│   ┌─────────────────┐                                        │
│   │ - ML 모델       │                                        │
│   │ - 외부 API      │                                        │
│   │ - 데이터베이스  │                                        │
│   └─────────────────┘                                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 Custom Skill 활용 예시

- **브랜드명 정규화**: "나이키", "NIKE", "nike" → "Nike"
- **제품 카테고리 분류**: 자체 ML 모델로 상품 분류
- **가격 추출**: 비정형 텍스트에서 가격 정보 파싱
- **외부 데이터 연동**: CRM, ERP 시스템과 연결

## 5. Knowledge Store

### 5.1 Knowledge Store란?

Skillset으로 생성된 **보강된 데이터를 Azure Storage에 저장**하는 기능입니다.  
검색 인덱스와는 **별도로** 데이터를 보관하여 다른 용도로 활용할 수 있습니다.

### 5.2 저장 형태

| 프로젝션 타입 | 저장 위치 | 활용 용도 |
|-------------|----------|----------|
| **Object Projection** | Blob Storage (JSON) | 데이터 백업, 다른 시스템 연동 |
| **Table Projection** | Table Storage | Power BI 분석, 보고서 |
| **File Projection** | Blob Storage (파일) | 추출된 이미지 저장 |

### 5.3 왜 Knowledge Store를 사용하는가?

```
┌─────────────────────────────────────────────────────────────────┐
│              Knowledge Store 활용 시나리오                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   [Skillset 실행]                                                │
│         │                                                        │
│         ├──→ [Search Index] → 검색 서비스에서 쿼리               │
│         │                                                        │
│         └──→ [Knowledge Store]                                   │
│                    │                                             │
│                    ├──→ Power BI로 대시보드 생성                  │
│                    ├──→ Azure ML로 추가 학습                      │
│                    ├──→ 다른 앱에서 JSON 데이터 활용              │
│                    └──→ 데이터 레이크로 아카이빙                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 6. 언제 Skillset을 사용해야 하는가?

### 6.1 ✅ Skillset이 필요한 경우

| 상황 | 이유 |
|-----|------|
| **이미지 또는 스캔된 문서** 검색 필요 | OCR로 텍스트 추출이 필수 |
| **다국어 콘텐츠**를 단일 언어로 검색 | 번역 스킬 활용 |
| **비정형 텍스트**에서 구조화된 정보 추출 | 엔티티 인식, 핵심구문 추출 |
| **감정/의견 분석**이 필요한 경우 | 감정 분석 스킬 |
| 데이터를 **검색 외 다른 용도로도 활용** | Knowledge Store로 데이터 보관 |
| **자동화된 메타데이터 생성** 필요 | 이미지 분석, 태그 추출 |

### 6.2 ❌ Skillset이 불필요한 경우

| 상황 | 대안 |
|-----|------|
| 이미 **정형화된 텍스트 데이터** | 바로 인덱싱 |
| **단순 키워드 검색**만 필요 | 기본 인덱싱 |
| **벡터 검색만** 필요 | 임베딩만 생성 (Text Split + Embedding 스킬은 가능) |
| **비용 민감** 환경 | AI Services 비용 발생 |
| **실시간 처리** 필요 | Skillset은 배치 처리에 적합 |

### 6.3 결정 플로우차트

```
                        [데이터 인덱싱 시작]
                               │
                               ▼
                    ┌─────────────────────┐
                    │ 이미지나 스캔된     │
                    │ 문서가 있는가?      │
                    └─────────────────────┘
                         │           │
                        Yes          No
                         │           │
                         ▼           ▼
                    ┌────────┐  ┌─────────────────────┐
                    │OCR 필요│  │ 텍스트에서 추가      │
                    │Skillset│  │ 정보 추출 필요?     │
                    │ 사용   │  └─────────────────────┘
                    └────────┘       │           │
                                    Yes          No
                                     │           │
                                     ▼           ▼
                              ┌──────────┐  ┌────────────┐
                              │Skillset  │  │ 기본       │
                              │사용      │  │ 인덱싱    │
                              └──────────┘  └────────────┘
```

## 7. 실제 비즈니스 시나리오

### 📋 시나리오 1: 법률/계약 문서 관리 시스템

**문제 상황**
- 수천 건의 스캔된 계약서 PDF
- 계약서 내 특정 조항, 회사명, 날짜 검색 필요
- 법률 검토를 위해 계약서 유형별 분류 필요

**Skillset 솔루션**
```
[스캔된 PDF] 
    → OCR (텍스트 추출)
    → Entity Recognition (회사명, 날짜, 금액 추출)
    → Key Phrase Extraction (핵심 조항 추출)
    → Custom Skill (계약서 유형 분류 ML 모델)
    → [검색 인덱스] + [Knowledge Store로 분석용 데이터]
```

**비즈니스 가치**
- 계약서 검색 시간 90% 단축
- 자동 분류로 법률팀 업무 효율 향상
- 만료 예정 계약 자동 알림 가능

---

### 📋 시나리오 2: 이커머스 상품 카탈로그

**문제 상황**
- 판매자가 올린 상품 이미지만 있고 설명이 부실
- 이미지 기반 검색 불가
- 상품 카테고리 자동 분류 필요

**Skillset 솔루션**
```
[상품 이미지]
    → Image Analysis (상품 설명 자동 생성)
    → OCR (이미지 내 텍스트: 브랜드명, 가격표 등)
    → Custom Skill (상품 카테고리 분류)
    → [검색 인덱스: 이미지 기반 텍스트 검색 가능]
```

**비즈니스 가치**
- 상품 발견율(Discoverability) 향상
- 판매자 등록 시간 단축
- 검색 정확도 개선으로 전환율 증가

---

### 📋 시나리오 3: 고객 피드백 분석 시스템

**문제 상황**
- 다국어로 작성된 수만 건의 고객 리뷰
- 제품별/기능별 피드백 분류 필요
- 불만 사항 우선 처리 필요

**Skillset 솔루션**
```
[고객 리뷰 (다국어)]
    → Language Detection (언어 감지)
    → Text Translation (영어/한국어로 통일)
    → Sentiment Analysis (긍정/부정/중립)
    → Entity Recognition (제품명, 기능 추출)
    → Key Phrase Extraction (주요 불만/칭찬 사항)
    → [검색 인덱스] + [Power BI 대시보드용 Knowledge Store]
```

**비즈니스 가치**
- 고객 불만 실시간 감지 및 대응
- 제품 개선 인사이트 자동 추출
- CS팀 우선순위 자동 분류

---

### 📋 시나리오 4: 의료 기록 검색 시스템

**문제 상황**
- 환자 기록이 PDF, 이미지(X-ray 판독문), 텍스트 혼합
- 환자명, 진단명, 처방약 검색 필요
- 개인정보 보호 규정 준수 필요

**Skillset 솔루션**
```
[의료 기록]
    → OCR (스캔된 기록에서 텍스트 추출)
    → Entity Recognition (환자명, 진단명, 약물명)
    → PII Detection (개인정보 탐지 및 마스킹)
    → Custom Skill (의료 코드 매핑: ICD-10)
    → [검색 인덱스: 권한 기반 검색]
```

**비즈니스 가치**
- 진료 정보 검색 시간 단축
- 규정 준수(Compliance) 자동화
- 환자 이력 추적 용이

---

### 📋 시나리오 5: 뉴스/미디어 아카이브

**문제 상황**
- 수십 년간 축적된 뉴스 기사, 영상 자막, 이미지
- 시대별, 인물별, 이슈별 검색 필요
- 영상에서 텍스트 추출 필요

**Skillset 솔루션**
```
[미디어 아카이브]
    → OCR (영상 자막, 신문 스캔)
    → Entity Recognition (인물, 조직, 장소, 날짜)
    → Key Phrase Extraction (주요 이슈)
    → Sentiment Analysis (기사 논조 분석)
    → [검색 인덱스] + [타임라인 분석용 Knowledge Store]
```

**비즈니스 가치**
- 아카이브 검색 효율 대폭 향상
- 팩트체크 및 연구 지원
- 트렌드 분석 데이터 제공

## 8. Skillset 도입 시 고려사항

### 8.1 비용 구조

| 항목 | 비용 발생 |
|-----|----------|
| Azure AI Search | 기본 서비스 비용 |
| Azure AI Services | 스킬 실행당 비용 (OCR, 분석 등) |
| Azure Storage | Knowledge Store 저장 비용 |
| Custom Skill 호스팅 | Azure Function 등 컴퓨팅 비용 |

> 💡 **팁**: 소규모 테스트는 Azure AI Services 무료 할당량으로 가능 (월 20회 인덱서 실행)

### 8.2 성능 고려사항

```
┌────────────────────────────────────────────────────────────┐
│                  Skillset 성능 최적화 팁                    │
├────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 📦 배치 크기 조절                                       │
│     - 기본 1,000개 문서 → 대용량은 조절 필요               │
│                                                             │
│  2. 💾 Enrichment Cache 활성화                              │
│     - 동일 문서 재처리 시 비용/시간 절약                    │
│                                                             │
│  3. 🎯 필요한 스킬만 사용                                   │
│     - 불필요한 스킬은 비용만 증가                           │
│                                                             │
│  4. 📐 적절한 Context 설정                                  │
│     - /document/* vs /document/pages/* 실행 횟수 차이       │
│                                                             │
│  5. 🔀 병렬 실행 가능한 스킬 설계                           │
│     - 의존성 없는 스킬은 병렬 처리                          │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

### 8.3 디버깅 도구

Azure Portal의 **Debug Sessions** 기능을 활용하면:
- Enrichment Tree 구조 시각화
- 각 스킬의 입출력 확인
- 오류 원인 파악

### 8.4 제한사항

| 제한 항목 | 값 |
|----------|-----|
| Skillset당 최대 스킬 수 | 30개 |
| 최대 Enrichment 깊이 | 10레벨 |
| OCR 이미지 크기 | 최대 50MB |
| 텍스트 입력 길이 | 스킬별 상이 (일반적으로 50,000자) |

## 📚 핵심 요약

### Skillset을 사용해야 하는 3가지 핵심 신호

1. **"검색하고 싶은 데이터가 텍스트가 아니다"**  
   → 이미지, 스캔 문서, PDF 등 → OCR 필요

2. **"원본 데이터에서 더 많은 정보를 추출하고 싶다"**  
   → 키워드, 엔티티, 감정 등 → NLP 스킬 필요

3. **"검색 외에 다른 분석/활용이 필요하다"**  
   → Power BI, ML 학습 등 → Knowledge Store 필요
