## 💡 핵심 개념: RRF (Reciprocal Rank Fusion)

### RRF 공식

```
RRF 점수 = 1/(k + rank_keyword) + 1/(k + rank_vector)
```

- **k**: 상수 (Azure AI Search에서 **60으로 고정**)
- **rank**: 각 검색 방식에서의 순위 (1부터 시작)

### RRF 계산 예시

| 문서 | 키워드 순위 | 벡터 순위 | RRF 점수 계산 | 최종 점수 |
|------|------------|----------|---------------|----------|
| A | 1 | 3 | 1/(60+1) + 1/(60+3) = 0.0164 + 0.0159 | **0.0323** |
| B | 3 | 1 | 1/(60+3) + 1/(60+1) = 0.0159 + 0.0164 | **0.0323** |
| C | 2 | 2 | 1/(60+2) + 1/(60+2) = 0.0161 + 0.0161 | **0.0322** |

→ **양쪽에서 골고루 높은 순위**를 받은 문서가 최종 상위로!

## ⚖️ 벡터 가중치 (Vector Weight)

```python
# vector_weight 파라미터로 비율 조절
vector_query = VectorizedQuery(
    vector=query_vector,
    fields="contentVector",
    k_nearest_neighbors=50,
    weight=0.7  # 벡터 검색에 70% 가중치
)
```

| weight 값 | 의미 | 사용 케이스 |
|-----------|------|-------------|
| **0.0** | 순수 키워드 검색 | 정확한 용어 매칭 필요 |
| **0.5** | 동등 가중치 (기본값) | 일반적인 검색 |
| **0.7~0.8** | 벡터 우선 | 자연어 검색이 많은 서비스 |
| **1.0** | 순수 벡터 검색 | 의미 기반 검색만 필요 |
